CodeSandbox and our platform

To save you some work, so you don't have to download and install an IDE (Integrated Development Environment), we use CodeSandbox, which is an online development environment that you can use for smaller projects. It comes with a lot of good features. Generally speaking IDE is a place/program/application where you can create your code and develop a website or an app.

JavaScript

This language can be found mainly in web development. You've probably heard of HTML and CSS before. HTML is the markdown code that we use to create web pages. It is only a static skeleton of the content, for example what is the title, what is the text, image, etc. A good analogy is that HTML can be seen as a dummy (plastic model of a person used in clothes shop for showing clothes). CSS adds styles to our HTML, with the help of CSS we can add colors, layout, etc. to our page. If the HTML is a dummy, then CSS is the clothes we put on it. And finally here's the javascript, it will help us add a little dynamics to our site. Animation, dynamic layout, etc. With Javascript, our dressed dummy changes into a dressed man who can move. JavaScript is a so-called interpreted language, which means that the interpreter that is built into your browser understands the individual commands that you write in the code.

Learn more about JavaScript from the official documentation

Basics

These informations are related to part 2 - Basics. Open the folder 2 in workshop folder.

fillRect()

In the script.js file you can see one function called draw(), at this moment we don’t really care about what exactly that means. Let's first take a look at what is written inside this function.

            
export function draw(c) {
    c.fillRect(20,20,200,150);
}

As the name suggests, this function draws a rectangle on the canvas. The letter c before the function is the name of the object that can draw on the canvas. The dot is a „connector“. In the language of programmers, this would mean something like: "I call the fillRect function on object c.“ Numbers in brackets are called parameters or arguments. Please note the semicolon at the end of the line. When the interpreter sees this semicolon it recognise that the line no longer continues. It's the same as the dot at the end of the sentence. When I change the value to: 

            
export function draw(c) {
    c.fillRect(200,100,300,50);
}

The rectangle shifts and resizes. You should now be able to determine what each number means. The first of these (200) is the number of pixels from the left edge, ie where we want to start our rectangle on the x-axis. The second of them (100) is the number of pixels from the top edge, ie where we want to start our rectangle on the y-axis. The third number is the width of the rectangle and the fourth is its height. It's important to realize that these axes are a little different then what we remember from math classes. In mathematics, the point [0,0] is located in the lower left corner. However, in the IT world, this point is in the upper left corner.

StrokeRect()

There are other methods we can use when painting on canvas. The strokeRect() method draws only the outline of the rectangle. 

        
export function draw(c) {
    c.strokeRect(200, 200, 300, 50);
}

What about adding some colors? Use this command to change the color of drawing objects.

            
export function draw(c) {
    c.fillStyle = "red";
    c.strokeStyle = "blue";
}

FillStyle and strokeStyle are not functions of object c, but its attributes. And we can assign a value to the attribute. In our case, it will be the name of the color in English. You can try any color that comes to your mind. Eg: pink, purple, crimson, salmon.

Comments

In programming languages we have the opportunity to add comments to our code. This makes easier to understand complicated parts or for example to add notes during learning. Commented lines start with // or placed between /* and */. The commented code won't get executed, it is only for the reader.

            
// this is an inline comment, the browser will ignore it

/*
    this is a multiline comment
    the browser will ignore the whole block
*/

Variables

Now we are going to get familiar with variables, types and operators. These are the really fundamentals in programming languages, not only in JavaScript, however their usage or syntax can slightly vary.

Declaration

A variable is a piece of memory where we can store a value. It is name-value pair and can be defined/declared with several different keywords. In Green Fox Academy Workshop we use the keyword let.

            
let variable;
let otherVariable;

Above we have created two variables, their names are variable and otherVariable respectively. Currently they have no value assigned. If you want to a assign a value to your variable, you will need to use the = operator:

            
let number = 5;
let name = "Fox";

Let's break it down now. The word let denotes a variable, followed by the name of the variable, then the = sign means that we want to store a value in this variable. The value depends on what is behind the equal sign. We can store many things in variables, more about this topic can be found here. It is good practice to name variables in a way that makes sense. The variable name should describe what the variable is used for.

Important: If you try to declare two variables with the same name (redeclaration), then you will get an error. In this case you need to ignore the let keyword for the second time or use another variable name.

After a variable has been declared you can change its value any time with the = sign:

Extra info: You can use const and var for variable declarations and all these 3 behaves differently. let is perfectly enough for us now, but you can read about them in the most referenced documentation, maintained by Mozilla and the community.

Types

Now after we know what are variables, lets examine what can be their values. As we've seen earlier, we have numbers for example. Is this enough for us? Nope, except if we want to create a calculator, but as the application gets more complicated we will need more types. In JavaScript we have 7 different primitive (basic) data types, but we will learn about only 3 of them:

  • number: any number, both integer and rational numbers
  • string: they represents any text or a single character
  • boolean: it has two values true or false

Boolean

Lets start with the simplest type, Boolean. It is a pretty common type and we have it in almost every language. It has only two values true or false.

            
let myBoolean = true;
let myOtherBoolean = false;
Logical operators

Operators which can be used with boolean values are called logical operators the most common ones are or (||) and and (&&). We consider these operators binary operators as they work with two values. There is also unary operators not (!).

To understand the logical and operator we can use a table

value1 value2 result
true true true
true false false
false true false
false false false

So in a nutshell logical and will result true only if both values are true

            
let result = true && true;
// result equals to true

let result2 = true && false;
//result2 equals to false

Logical or is pretty similar to the and, but its result is always true if at least one operand is true.

value1 value2 result
true true true
true false true
false true true
false false false 
            
let result = true || true;
// result equals to true

let result2 = true || false;
//result2 equals to true

let result3 = false || false || true || false;
// result3 equals to true

let result4 = false || false;
// result4 equals to false

not is a unary operator which means it has only one operand. It simply negates the value.

            
let result = !true;
// result equals to false

let result2 = !false
// result2 equals to true

What do you think what will be the result of !!true? Lets try it and find out why did you get that result.

Assignement operators

With the equals operator (===) you can check if the operands are equals or not. This operator can be used with all the other types but will always result a boolean.

            
let result = true === true;
// result equals to true

let result2 = true === false;
// result2 equals to false

let result3 = false === false;
// result3 equals to true

let result4 = 5 === 5;
// result4 equals to true

Important: there is another equals operator == which works almost the same way, but in some cases they will have different result. As a good practice we encourage everybody to use the triple variant.

The not equals operator (!==) is the counterpart of the equals, you can check with it whether two values are different or not.

            
let result = true !== true;
// result equals to false

let result2 = true !== false;
// result2 equals to true

let result3 = 1 !== 2;
// result3 equals to true

Numbers

Numbers represents anything between -Infinity and Infinity and you can do any operation with them what you already did in the elementary school.

Extra info: Some languages have specific types for rational numbers but in JavaScript there is only number, regardless to its value.

Mathematical operators

Just like in math you can do different operations with numbers. Most of them you know from your elementary school times.

  • Addition +
  • Substraction -
  • Multiplication *
  • Division /
  • Modulo %
The first three of these operators (+,-,*) work as you expect them to work. You can try in out in a console. 

            
let addition = 5 - 6;
// addition equals to -1

let substraction = 5 - 6;
// substraction equals to -1

let multiplication = 5 * 6;
// multiplication equals to 30

Division is almost the same too, but there is a slight difference. In math, you cannot divide by zero. In JavaScript you can get a strange result.

            
let result = 10 / 4;
// result is 2.5

let result2 = 10 / 0;
// result2 is Infinity

Modulo is less known but we do have it in math too. With modulo you can get the reminder of a division. It is quite important operator in programming world.

            
let reminder = 5 % 3;
//reminder is 2;

Strings

Strings represents any text or character in programming languages, you just need to use quotation marks such as ' or ".

            
let myString = 'This is a string';
let anotherString = "This is also a string";

We have a special form of strings as well, called string template literal. We use them, when we want to make something dynamic. For this you will need to use some more special characters:

            
let name = 'Mark';
let greeting = `Hello ${name}!`;
// greeting equals to "Hello Mark!"

Important: Mind the special characters, we used the backtick ` instead of simple quotation marks and surrounded the variable with ${...}

Operators

There are several operation what can be performed on strings but we will focus only on two of them. First of them is concatenation (+) and second is .length operator.

With the + sign you can append/concatenate a string to another.

            
let greetWorld = 'Hello ' + 'World!';
//greetWorld equals to "Hello World!"

let name = 'Dave';
let greeting = 'Hello ' + name + '!';
// greeting equals to "Hello Dave!"

Important: As you can see the result is the same as with string template literals, these template literals are pretty new in JavaScript and preferred over the concatenation.

Strings in JavaScript have a property, called length. With this property you can get the count of characters in the string.

            
let charCount = 'Hello'.length;
// charCount equals to 5

Colors

Having only black and white drawing can be kind of boring. We can change the color of our future drawings by changing the property strokeStyle or fillStyle. These properties belong to the context c defined in canvas. The default color is black, once you change the color all of the next drawings are using this color, untill you change it again. strokeStyle changes the color of the outline of a rectangle. fillStyle changes the fill color. We can simply use the english names of colors. The list of possible colors is really long.

            
// This rectangle is going to be black
c.fillRect(50,50,100,100);

// Now we change the color to red
c.fillStyle = "red";
// This one is going to be red
c.fillRect(50,50,100,100);

// Same for just the outline (stroke)
c.strokeRect(50,50,100,100);
c.strokeStyle = "red";
c.strokeRect(50,50,100,100);

RGB notation

The list of colors whose names we can use right away is long enough, but it certainly lacks some colors. There is another way how we can represent a color in JavaScript - RGB. RGB color values are supported in all browsers. An RGB color value is specified with: rgb(red, green, blue). Each parameter (red, green, and blue) defines the intensity of the color as an integer between 0 and 255. For example, rgb(0, 0, 255) is rendered as blue, because the blue parameter is set to its highest value (255) and the others are set to 0. 

            
// Setting color to red
c.fillStyle = "rgb(255,0,0)";

// Setting color to blue
c.fillStyle = "rgb(0,255,0)";

// Setting color to green
c.fillStyle = "rgb(0,0,255)";

// Setting color to purple - 50% of red, 0% of green and 50% of blue
c.fillStyle = "rgb(128,0,128)";

RGBA notation

In addition to RGB notation, we can also use RGBA notation, the last parameter indicates the opacity of the object. Value 1 means opacity and value 0 means absolute transparency.

            
// Red color with 50% opacity
c.fillStyle = "rgba(255,0,0,0.5)

Because both of these notations are strings we can create them using string template literal or concatenation and include some variables.

            
let opacity = 0.1;
c.fillStyle = `rgba(0,0,0, ${opacity * 3})`;

Conditions

You can make a piece of your code optional, with the if statement. It lets you define a condition which your code depends on

            
let myVariable = false;
if (myVariable === true) {
    console.log('The condition has been evaluated to true');
}

If you run the above code in the console or in a webpage you will see nothing in the console. Please change something to see the desired message in the console

Our if statement gives us more capabilities, we can tell what we want to do if the given condition has been evaluated to false.

            
let a = 7;
if (a < 5) {
    console.log(a / 2);
} else {
     console.log(a * 2);
}

You can read more about the if statement in the documentation.

Loops

Sometimes you want to execute a piece of code several times. Like in the real life when you want to move books from one shelf to another.

  • you grab the book
  • you move it to the other shelf
  • you release the book
And you do this until you have no books to move. For example if you have 10 books: 

            
let bookCount = 10;

for (let i = 0; i < bookCount; i++) {
    // here comes the code which moves the books
    // this block will be repeated bookCount times
}

For indicates the beginning of a new for-loop, followed by the initialization of the variable i, the condition, and the change of the variable. The most important part of for-loop for us is the condition. If the condition is true, the commands in curly brackets are going to be executed.

Printing the number of current book is also fairly easy. We call the number of a loop (book in our case) an index

            
for (let i = 0; i < 10; i++) {
    console.log(i);
}

We already know what parts the for-loop consists of. But what exactly that change part means?

            
let i = 0;
// These 3 statements means exactly the same - addition of 1 to current value of i
i++;
i=i+1;
i+=1;

// Same with substraction
i--;
i=i-1;
i-=1;

// If we want to add more then one, then there are only 2 options
i+=10;
i=i+10;
i-=10;
i=i-10;

Drawing

We already know how to draw a rectangle or its outline. We can also draw any other shape by drawing single lines. Following example will draw a triangle.

            
export function draw(c) {
    c.beginPath();
    c.moveTo(100, 30);
    c.lineTo(150, 115);
    c.lineTo(50, 115);
    c.lineTo(100, 30);
    c.stroke();
    c.fill();
}

Imagine that there is an invisible hand that draws on the canvas for you. We have to start drawing with the beginPath function, which wakes the invisible hand to life. With the moveTo function we can move the hand to a specific place. Finally, the invisible hand begins to draw with invisible ink on the canvas with thelineTo function. Once it's done, we call the stroke function and the ink appears. This is how we can draw a triangle. Using the fill method instead of the stroke method, we can fill the object with color, but only if the object is closed.

Functions

Usually there are many repeating code blocks in an application, when we want to execute the same code but with different circumstances/environments. Just imagine how you pay your bills. The steps are the same you just transfer different amount to different bank accounts. So the amount and the bank account are parameters of this activity. In programming languages these activities are called functions.

            
function payBill(amount, beneficiary) {
    // transfer amount money to the beneficiary
};

Within functions you can use the return keyword which will give back the function's output. For example if you want to add 2 numbers, then you will need the result as well:

            
function add(a, b) {
    return a + b;  // the return keyword will give you back the result
};

console.log(add(1, 2)) // will print 3

Another example of function for drawing a square. Squares are rectangles with width being equal to height.

            
export function draw(c) {
    function strokeSquare(left, top, size) {
        c.strokeRect(left, top, size, size);
    }
    strokeSquare(100,100,200);
}

The keyword function means that we are starting to declare a new function. This is followed by the name of this function, its parameters in brackets, and the body of the function in curly brackets. In our case we want to draw squares, 3 parameters are enough for that. left - how much from the left edge. top - how much from the top edge. size - what should be the size of the square. In the body of the function, we only pass these parameters to the already known strokeRect function.

Events

So far you can draw on canvas with the help of variables, conditions and loops. With the help of HTML and CSS you can create nice dynamic website. This sounds cool, but still not enough. The user might want to interact with your webpage. Do actions like clicking on a button, save some data during sign up or simply like/share some content. This is where we will need the events. Events are actions made by the user and we want to respond to them somehow. Let's see how events work on an example. We want to create a program that will draw foxes on the canvas when moving the mouse over the canvas. To do this, we will need a method for drawing foxes. Let the method be:

            
export function draw(c) {
    function drawFox(x, y) {
        let size = 20;

        c.beginPath();
        c.moveTo(x, y);
        c.lineTo(x+size/7*2, y-size/7*3);
        c.lineTo(x+size/7*3, y-size/7);
        c.lineTo(x+size/7*4, y-size/7);
        c.lineTo(x+size/7*5, y-size/7*3);
        c.lineTo(x+size, y);
        c.lineTo(x+size/2, y+size/7*3);
        c.lineTo(x, y);
        c.fill();
    }
}

We have a method, now we have to tell JavaScript when to use this method. First, we select the part of the screen that contains our canvas. In HTML we can find that this part uses an id named canvas. 

            
let canvas = window.document.getElementById("canvas");

Now we need to add event listener to this canvas.

            
canvas.addEventListener('mousemove', function(e) {
    // Here we will put the code that is going to be executed, when the event is triggered
});

To this function we can add whatever we want to happen if the mouse is moved. This is the reason of the name of this event – mousemove.

            
canvas.addEventListener("mousemove", function (e) {
    drawFox(e.clientX, e.clientY);
});

e.clientX and e.clientY are coordinates of the mouse when the event happened. Full code of this example can be found in part 10 of the workshop.
